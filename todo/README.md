# todo/

The project's own to-do, one file per week. This is the work *we* do (Bear, the gamers, the agent) — distinct from the gamer-facing assignment in `unity/gate-pilot/`.

## Convention

- One file per week: `week-NN.md`, NN zero-padded. The date range lives in the file header.
- The **current week** is the highest-numbered file.
- Past weeks stay as the run history — don't delete them (SNICKERDOODLE: archive, never delete). A finished week is the record of what actually happened.
- A checkbox is checked only when its done-condition is met. Where a task is a **gate**, "looks done" doesn't clear it — the named owner does (P4).
